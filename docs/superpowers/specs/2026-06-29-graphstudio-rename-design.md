# GraphStudio Rename ‚Äî Test Suite + Rename Design

**Date:** 2026-06-29  
**Status:** Approved

---

## Overview

Rename the product from "SmartStudio" to "GraphStudio" across the Rust backend, React frontend, and MCP server. Before touching any display text, build a full test suite that (a) covers critical endpoints, and (b) acts as a precision checkpoint ‚Äî canary tests go GREEN now and turn RED exactly when their guarded item is renamed, confirming the rename landed without side-effects.

**Stack:**
- Rust/Axum backend (`server/`) ‚Äî Axum 0.8, port 3001
- React frontend (`src/`) ‚Äî Vite + Zustand + TanStack
- MCP server (`mcp-server/`) ‚Äî standalone Node.js package

## Phase 1 ‚Äî Rust Integration Tests

### Infrastructure

**Key challenge:** `AppState` requires real SQLite files. Chosen approach: **temp-dir fixtures**.

Each test suite creates a `TempDir`, writes a minimal `environment.toml` pointing `home_path` there, calls the real `instance_config::resolve()/ensure_ready()` chain to create the SQLite schema, constructs `AppState`, and wraps the router with `axum-test`'s `TestServer`. No mocking ‚Äî full fidelity.

**Required production code change:** Extract the router definition from `main()` into:
```rust
pub fn build_router(state: Arc<AppState>) -> Router { ... }
```
This is a pure mechanical extraction ‚Äî no logic change.

**New dev-dependencies in `server/Cargo.toml`:**
```toml
[dev-dependencies]
axum-test = "17"
tempfile = "3"
```
(`tokio`, `serde_json`, `uuid` are already dependencies.)

### Test helper (`tests/common/mod.rs`)

```
pub async fn setup_server() -> (TestServer, TempDir)
```

Steps:
1. Create `TempDir`
2. Write minimal `environment.toml` (`is_new = true`, `home_path = <tmp>`, `client = "test"`, `app_type = "test"`, `environment = "test"`)
3. Call `instance_config::load() ‚Üí resolve() ‚Üí ensure_ready()` ‚Äî creates SQLite schema
4. Open `db::Database` and `agent::db::AgentDb`
5. Seed agent pricing/model config
6. Construct `AppState` with all in-memory fields (`Arc::new(...)`, `None` defaults for optional fields)
7. Call `build_router(Arc::new(state))`
8. Return `TestServer::new(router)` + `TempDir` (caller holds TempDir to prevent early cleanup)

### Test files

**`tests/test_health.rs`**
- `GET /api/health` ‚Üí 200
- `GET /api/identity` ‚Üí 200, body has `id`, `client`, `app_type`, `environment`, `display_name`

**`tests/test_crud.rs`** ‚Äî happy path (create + read) + 404 on missing ID:
- `POST /api/pipelines` + `GET /api/pipelines/{id}`
- `POST /api/sources` + `GET /api/sources/{id}`
- `POST /api/dataviews` + `GET /api/dataviews/{id}`
- `POST /api/connections` + `GET /api/connections/{id}`
- `POST /api/modules` + `GET /api/modules/{id}`
- `POST /api/dimensions` + `GET /api/dimensions/{id}`
- `POST /api/filter-configs` + `GET /api/filter-configs/{id}`
- `POST /api/templates` + `GET /api/templates`

Each test POSTs a minimal valid body, asserts 200/201, then GETs by the returned ID, asserts the `display_name` round-trips. Error case: GET with a nonexistent ID ‚Üí 404.

**`tests/test_generate.rs`**
- Create a minimal DataView via the API
- `POST /api/generate/dataview/{id}/preview` ‚Üí 200, body has `"files"` key containing exactly: `proto/<name>.proto`, `Cargo.toml`, `build.rs`, `src/main.rs`, `src/service.rs`, `src/rest.rs`
- `POST /api/generate/dataview/{id}/write` ‚Üí 200, body has `"files_written"` array, AND each path in the array exists on disk

**`tests/test_canaries.rs`** ‚Äî integration canaries (GREEN now, RED when renamed):
- Bundle `Content-Disposition`: `POST /api/bundle/export` with empty selection ‚Üí response `Content-Disposition` header contains `"smartstudio-bundle-"`
- SQLite filename: After `setup_server()`, `state.db_path` ends with `"smartstudio.db"`

**`tests/test_template_canaries.rs`** ‚Äî template string canaries (GREEN now, RED after template rename):
- Create a DataView, call preview, assert `files["proto/<name>.proto"]` contains `"Generated from SmartStudio"`
- Same call, assert `files["src/service.rs"]` contains `"Generated by SmartStudio"`

### Inline unit-test canaries (inside source files)

These live in `#[cfg(test)] mod tests` blocks within the source file ‚Äî no AppState needed.

| File | Test name | Assertion |
|---|---|---|
| `src/instance_config.rs` | `namespace_dir_is_smartstudio` | `NAMESPACE_DIR == "smartstudio"` |
| `src/pg_pools.rs` | `pg_max_concurrency_env_var_name` | Setting `SMARTSTUDIO_PG_MAX_CONCURRENCY` env var is picked up by `default_max_size()` |
| `src/graph/parity.rs` | `bealls_duckdb_env_var_name` | `env_duckdb_path()` reads `SMARTSTUDIO_BEALLS_DUCKDB`; set it, assert the value is returned |

For `pg_pools` and `parity`, the private functions need to be made accessible to their own `#[cfg(test)]` blocks (they already are by default since the test is in the same module).

## Phase 2 ‚Äî Frontend Tests (Vitest)

### Infrastructure

Add to root `package.json` devDependencies:
```
vitest, @vitest/coverage-v8
@testing-library/react, @testing-library/jest-dom, @testing-library/user-event
jsdom
```

Add `test` config block to `vite.config.ts`:
```ts
test: {
  environment: 'jsdom',
  setupFiles: ['./src/setupTests.ts'],
  globals: true,
}
```

Create `src/setupTests.ts`:
```ts
import '@testing-library/jest-dom'
```

Add `"test": "vitest run"` to `package.json` scripts.

### Test files

**`src/__tests__/App.smoke.test.tsx`**
- Renders `<App />` with mocked `api.getIdentity()` returning a minimal Identity object
- Asserts no crash, sidebar element present in DOM

**`src/__tests__/WorkspaceLayout.canary.test.tsx`** ‚Äî GREEN now, RED after rename:
- Renders `<WorkspaceLayout>` with minimal props + mocked identity
- Asserts element with text `"SmartStudio"` is in document

**`src/__tests__/AgentApp.canary.test.tsx`** ‚Äî GREEN now, RED after rename:
- Renders agent `<App />` with mocked API
- Asserts heading containing `"SmartStudio Agent"` is present

**`src/__tests__/CoreServiceWorkspace.canary.test.tsx`** ‚Äî GREEN now, stays GREEN (not renamed):
- Renders `<CoreServiceWorkspace>` with mocked state
- Asserts placeholder text contains `"smartstudio-data"`

### MCP server canary

**`mcp-server/src/__tests__/canary.test.ts`** ‚Äî GREEN now, stays GREEN (not renamed):
